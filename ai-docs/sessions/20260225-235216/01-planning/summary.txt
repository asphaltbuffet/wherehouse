The refactoring centralizes all config write operations (init, set, unset, check) into a new `internal/config/writer.go` package, removing direct TOML manipulation from `cmd/config/` and leaving the CLI layer as thin orchestration calling library functions. A golden file (`internal/config/testdata/config.golden`) rendered via `text/template` replaces the hand-built `marshalConfigWithComments` string builder, serving as both the `config init` template and a regression fixture. The known `TestInitThenCheck` bug is fixed by adding `toml` struct tags alongside the existing `mapstructure` tags on all `Config` sub-structs, enabling `go-toml/v2` to correctly deserialize snake_case TOML keys.
