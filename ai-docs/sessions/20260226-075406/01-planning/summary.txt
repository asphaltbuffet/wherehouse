The plan adds a pre-flight `os.Stat` check inside `cli.OpenDatabase` that surfaces a clear "run `wherehouse init`" error before any SQLite driver is invoked, replacing the misleading "not enough memory" failure. A new `cmd/init` package implements `wherehouse init [--database <path>] [--force]` which creates the database (running migrations and seeding system locations), with best-effort backup of any existing file when `--force` is used. Four files change: `internal/cli/database.go` (sentinel error + check), and three new files under `cmd/init/` (doc, init, wired into root.go).
