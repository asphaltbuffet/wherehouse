The `list` command is a read-only tree-rendering command structured as `cmd/list/` following the exact pattern of `cmd/move/`, with three files: `list.go` (cobra command + runList + node builders), `output.go` (ASCII tree rendering + JSON structs), and `helpers.go` (thin wrappers). Only one new database function is required (`GetRootLocations` in `internal/database/location.go`); all other DB access reuses existing `GetLocationChildren` and `GetItemsByLocation`. Tree construction uses Go-level recursion over existing DB functions (not SQL WITH RECURSIVE), producing a `LocationNode` in-memory tree that renders as ASCII box-drawing connectors for human output or a nested JSON array for `--json` mode.
