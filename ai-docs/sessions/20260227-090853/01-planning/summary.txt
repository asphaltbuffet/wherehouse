The `found` command reuses the existing `item.found` event type (already implemented in itemEventHandler.go) and, when --return is given, chains it with a second `item.moved` (rehome) event - no new event types or database layer changes are needed. The package structure mirrors cmd/move/ with found.go, item.go, helpers.go, and doc.go; all selector and location resolution delegates to internal/cli as in the move command. Seven gaps require user clarification before implementation: most critically, the behavior when an item is not at Missing, how to infer home when no TempOriginLocationID exists, and whether project association flags belong on the --return path.
