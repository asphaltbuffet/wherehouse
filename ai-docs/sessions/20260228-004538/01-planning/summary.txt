The existing cmd/move tests correctly apply the thin-entrypoint pattern by testing moveItem() directly, but have three critical gaps: zero flag-to-parameter wiring tests, zero output routing tests, and six testifylint violations (assert.Contains(err.Error()) instead of assert.ErrorContains). The plan adds a NewMoveCmd(fn moveItemFunc) constructor to create a testability seam, then adds ~11 new tests covering wiring (--temp, --project, --keep-project, --note), output routing (--json, human, --quiet), and error propagation (invalid project, unresolvable selector/destination), while deleting one dead t.Skip test and modifying six existing tests to fix linting violations and remove string-literal assertions.
