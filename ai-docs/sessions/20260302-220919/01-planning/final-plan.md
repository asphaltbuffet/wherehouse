# Final Implementation Plan: EventType Typed Enum Refactor

Date: 2026-03-02

## Overview

Refactor `EventType string` on the `Event` struct to `EventType EventType` (the typed `int` enum from `internal/database/eventTypes.go`). The database column remains a string. The `stringer` tool (run via `mise run generate`) generates `String()` methods. A map-based `ParseEventType` using `.String()` calls on each constant provides the reverse mapping — the stringer comments remain the single source of truth. Custom `driver.Valuer` / `sql.Scanner` implementations bridge the typed int to its string representation in the database layer.

---

## Clarifications Applied

1. `ParseEventType` lives in `eventTypes.go` alongside the constants (all enum logic in one file, not a separate file).
2. Mock update: add a `//go:generate mockery` directive to `cmd/move/mover.go`; hand-update the mock body since mockery cannot be run mid-session.
3. `stringer` is already available via `mise run generate` — include `mise run generate` as an explicit step; do NOT add stringer as a new dependency.
4. TDD order is explicit: write failing test → implement → confirm passing test, for every change.
5. Work is split into parallel batches by agent scope.
6. **`ParseEventType` uses a map-based reverse lookup** — no string literals. The package-level map is initialized using `.String()` calls on each constant so the stringer `// linecomment` annotations remain the single source of truth. Because the map initialization calls `.String()`, `eventTypes_string.go` (the generated file) MUST exist before any code calling `ParseEventType` is compiled. This means `mise run generate` must be run as part of the db-developer batch BEFORE other code compiles.

## Architecture

### Layer 1: Stringer Directive (db-developer)

Add to the top of `internal/database/eventTypes.go`:

```go
//go:generate stringer -type=EventType -linecomment
```

The `-linecomment` flag makes `ItemCreatedEvent.String()` return the inline comment value `"item.created"` rather than the Go identifier `"ItemCreatedEvent"`. Run with `mise run generate`.

Generated output: `internal/database/eventTypes_string.go` (do not edit manually).

### Layer 2: ParseEventType (db-developer)

Add to `internal/database/eventTypes.go` (alongside the constants):

```go
// eventTypeByName maps string representations back to EventType constants.
// Initialized using .String() on each constant so the stringer linecomments
// remain the single source of truth — no separate string literals to maintain.
var eventTypeByName = map[string]EventType{
    ItemCreatedEvent.String():      ItemCreatedEvent,
    ItemMovedEvent.String():        ItemMovedEvent,
    ItemMissingEvent.String():      ItemMissingEvent,
    ItemBorrowedEvent.String():     ItemBorrowedEvent,
    ItemLoanedEvent.String():       ItemLoanedEvent,
    ItemFoundEvent.String():        ItemFoundEvent,
    ItemDeletedEvent.String():      ItemDeletedEvent,
    LocationCreatedEvent.String():  LocationCreatedEvent,
    LocationRenamedEvent.String():  LocationRenamedEvent,
    LocationMovedEvent.String():    LocationMovedEvent,
    LocationDeletedEvent.String():  LocationDeletedEvent,
    ProjectCreatedEvent.String():   ProjectCreatedEvent,
    ProjectCompletedEvent.String(): ProjectCompletedEvent,
    ProjectReopenedEvent.String():  ProjectReopenedEvent,
    ProjectDeletedEvent.String():   ProjectDeletedEvent,
}

// ParseEventType converts a string representation to an EventType constant.
// Returns an error for unrecognized strings to fail loudly on mismatch.
func ParseEventType(s string) (EventType, error) {
    if et, ok := eventTypeByName[s]; ok {
        return et, nil
    }
    return 0, fmt.Errorf("unknown event type %q", s)
}
```

**Critical compile-order constraint**: `eventTypeByName` calls `.String()` on each constant. The `.String()` method is generated by stringer into `eventTypes_string.go`. This file must exist before the package compiles. Therefore `mise run generate` must be run before any other work in this session proceeds past the db-developer batch.

### Layer 3: Database Bridge Value/Scan (db-developer)

New file: `internal/database/eventTypes_sql.go`

```go
package database

import (
    "database/sql/driver"
    "fmt"
)

// Value implements driver.Valuer, persisting EventType as its string representation.
func (e EventType) Value() (driver.Value, error) {
    return e.String(), nil
}

// Scan implements sql.Scanner, reading a string from the database and converting
// back to the typed EventType constant.
func (e *EventType) Scan(src any) error {
    s, ok := src.(string)
    if !ok {
        return fmt.Errorf("EventType.Scan: expected string, got %T", src)
    }
    parsed, err := ParseEventType(s)
    if err != nil {
        return err
    }
    *e = parsed
    return nil
}
```

## Key Constraints

1. `stringer -linecomment` flag is mandatory: without it `ItemCreatedEvent.String()` returns `"ItemCreatedEvent"` not `"item.created"`.
2. `mise run generate` MUST be run during the db-developer batch (Step A2) before `ParseEventType` or any code that calls it is compiled. The `eventTypeByName` map uses `.String()` which requires the generated file to exist.
3. `ParseEventType` is map-based: adding a new enum value requires adding one entry to `eventTypeByName`. No string literals to maintain separately.
4. `insertEvent` in `helper_test.go` is test-only but in the `database` package; it must also accept `EventType`.
5. No database migration is needed: column type and stored string values are unchanged.
6. `internal/styles` must not import `internal/database`; the `.String()` call at the boundary preserves this.
7. Mocks: add `//go:generate mockery --name=moveDB` directive to `cmd/move/mover.go`; hand-update the mock body to compile with the new signature. Do not attempt to run mockery mid-session.

## Trade-offs and Alternatives

| Alternative | Verdict |
|-------------|---------|
| Keep `AppendEvent` as string, only type the struct field | Rejected: callers still use magic strings; no compile-time safety |
| `type EventType = string` alias | Rejected: no real type safety |
| Move `ParseEventType` to a separate file | Rejected per clarification: keep all enum logic in `eventTypes.go` |
| Update `EventStyle` to accept `EventType` | Rejected: would introduce `styles` → `database` dependency; use `.String()` instead |
| Store integers in the database | Rejected: requirement specifies string storage must be preserved |
| `ParseEventType` as switch with string literals | Rejected per clarification #5: duplicates the string values that stringer linecomments already define; use map initialized via `.String()` instead |
